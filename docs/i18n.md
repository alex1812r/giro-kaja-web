# Internationalization (i18n)

Same approach as `giro-kaja-app`: **i18next** + **react-i18next**.

## Locales

| Code | File | Role |
|------|------|------|
| `es` | `src/shared/locales/es.json` | Default |
| `en` | `src/shared/locales/en.json` | English |

JSON files are based on the mobile app locales, plus web keys (`brand`, `shell`, `forgotPassword`, `resetPassword`, `currency`, login extras).

## Runtime

- Init: `src/shared/i18n/config.ts`
- Provider: `I18nProvider` inside `AppProviders`
- Persist: `localStorage` key `giro-kaja:locale`
- Default / fallback: **`es`**
- Switch UI: Settings → language buttons (`setLocale`)
- Hydration: i18n always boots in `es` (SSR = first client paint). Stored locale applies in `useEffect` after mount.

## Usage

```tsx
import { useTranslation } from "react-i18next";

const { t } = useTranslation();
t("tabs.home");
t("login.submit");
```
